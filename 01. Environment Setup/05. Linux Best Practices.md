# Introduction

Linux is the operating system you'll spend most of your time using as a penetration tester.

Whether you're attacking a target, enumerating services, transferring files, or escalating privileges, you'll constantly interact with Linux systems.

Being comfortable with the Linux command line is one of the most valuable skills you can develop.

---

# Why Learn Linux?

Nearly every cybersecurity tool is built for Linux.

Examples include:

- Nmap
- Metasploit Framework
- Burp Suite
- Gobuster
- SQLMap
- Hydra
- John the Ripper
- Wireshark
- Netcat

Understanding Linux also helps when attacking Linux servers during real penetration tests.

# Working with Files

Create a file:

```bash
touch notes.txt
```

Create directories:

```bash

```

Create nested directories:

```bash

```

Copy files:

```bash

```

Copy directories:

```bash
cp -r folder backup
```

Move files:

```bash
mv file.txt Documents/
```

Rename files:

```bash
mv old.txt new.txt
```

Delete file:

```bash
rm file.txt
```

Delete directory:


# Searching for Files

Find files:

```bash
find / -name "password.txt"
```

Find directories:

```bash
find / -type d
```

Search inside files:

```bash
grep "password" config.txt
```

Recursive search:

```bash
grep -R "root" .
```

Locate executable:

```bash
which python3
```

# Permissions and Ownership

View permissions:

```bash
ls -l
```

Change permissions:

```bash
chmod 755 script.sh
```

Make executable:

```bash
chmod +x exploit.sh
```

Change owner:

```bash
sudo chown user:user file.txt
```

Permission example:

```
-rwxr-xr-x
```

Meaning:

```

Read
Write
Execute

Group

Read
Execute

Others

Read
Execute
```

Numeric permissions:

| Value | Permission |
|--------|------------|
| 7 | rwx |
| 6 | rw- |
| 5 | r-x |
| 4 | r-- |

# SSH Best Practices

Connect:

```bash
ssh user@192.168.1.10
```

Specify private key:

```bash
ssh -i id_rsa user@target
```

Copy files:

```bash
scp file.txt user@host:/tmp
```

Generate SSH key:

```bash
ssh-keygen
```

Always verify the target host before accepting its fingerprint.

# Common Mistakes

Avoid these common beginner errors:

- Working as root unnecessarily.
- Using `rm -rf` without checking the path.
- Forgetting to update the system.
- Ignoring error messages.
- Not verifying downloaded scripts.
- Running unknown commands copied from the Internet.
- Saving important files in `/tmp`.
- Forgetting to take snapshots before major changes.

---

# Best Practices During an Engagement

A professional penetration tester should:

- Keep systems updated.
- Organize project files.
- Use descriptive filenames.
- Keep notes throughout the assessment.
- Verify every command before execution.
- Minimize unnecessary changes on target systems.
- Understand each tool before using it.
- Follow a structured methodology instead of guessing.

Remember:

> Good methodology beats memorizing hundreds of commands.

# Summary

Linux is the foundation of modern penetration testing.

By mastering the command line, filesystem navigation, permissions, networking, and process management, you'll become faster and more efficient during every phase of a penetration test.

The goal is not to memorize every command, but to understand how Linux works and use it confidently.
