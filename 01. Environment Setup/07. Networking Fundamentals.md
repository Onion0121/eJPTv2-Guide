# Introduction

Networking is one of the most important topics in penetration testing.

Before exploiting a target, you must understand how computers communicate, how data travels across networks, and how services are exposed.

Almost every eJPT machine begins with network enumeration.

---

# Why Networking Matters

A penetration tester constantly works with:

- IP addresses
- Ports
- Network services
- Firewalls
- Routers
- DNS
- Network protocols

Understanding these concepts allows you to:

- Discover hosts
- Enumerate services
- Identify attack surfaces
- Exploit exposed applications
- Pivot through internal networks

---

# The OSI Model

The **OSI (Open Systems Interconnection)** model describes how data travels across a network.

| Layer | Name | Example |
|--------|------|---------|
| 7 | Application | HTTP, FTP, DNS |
| 6 | Presentation | SSL/TLS |
| 5 | Session | NetBIOS |
| 4 | Transport | TCP, UDP |
| 3 | Network | IP |
| 2 | Data Link | Ethernet |
| 1 | Physical | Cable, Fiber, Wi-Fi |

As a penetration tester, you'll mainly interact with:

- Layer 7 (Applications)
- Layer 4 (TCP/UDP)
- Layer 3 (IP)

---

# The TCP/IP Model

The Internet uses the TCP/IP model.

| Layer | Protocols |
|--------|-----------|
| Application | HTTP, FTP, DNS, SSH |
| Transport | TCP, UDP |
| Internet | IP, ICMP |
| Link | Ethernet, Wi-Fi |

This simplified model is what you'll encounter most often.

---

# IP Addresses

Every device connected to a network has an IP address.

Example:

```
192.168.1.10
```

An IP address identifies a host so other devices know where to send data.

---

# Public vs Private IP Addresses

## Public IP

Accessible from the Internet.

Example:

```
8.8.8.8
```

---

## Private IP

Used inside local networks.

Ranges:

```
10.0.0.0/8

172.16.0.0 - 172.31.255.255

192.168.0.0/16
```

These addresses cannot be reached directly from the Internet.

Most eJPT labs use private IP addresses.

---

# IPv4 Structure

IPv4 consists of **32 bits** divided into four octets.

Example:

```
192.168.56.101
```

Each number ranges from:

```
0 - 255
```

---

# Subnet Masks

A subnet mask separates:

- Network portion
- Host portion

Example:

```
IP

192.168.1.50

Mask

255.255.255.0
```

This means every device in:

```
192.168.1.x
```

belongs to the same subnet.

---

# CIDR Notation

Instead of writing:

```
255.255.255.0
```

You will often see:

```
/24
```

Examples:

| CIDR | Subnet Mask |
|-------|-------------|
| /8 | 255.0.0.0 |
| /16 | 255.255.0.0 |
| /24 | 255.255.255.0 |

Example:

HTTP

---

# TCP vs UDP

## TCP

- Connection-oriented
- Reliable
- Error checking
- Ordered packets

Examples:

- HTTP
- HTTPS
- SSH
- FTP
- SMB

---

## UDP

- Connectionless
- Faster
- No guarantee of delivery

Examples:

- DNS
- SNMP
- DHCP
- TFTP

---

# The Three-Way Handshake

TCP establishes connections using three steps.

```
Client
   |
SYN
   |
Server
   |
SYN-ACK
   |
Client
   |
ACK
```

Once complete, communication begins.

---

# MAC Addresses

Every network interface has a unique MAC address.

Example:

```
00:50:56:A3:92:FC
```

MAC addresses operate at **Layer 2**.

---

# ARP

ARP (Address Resolution Protocol) maps IP addresses to MAC addresses.

View the ARP cache:

```bash
arp -a
```

or

```bash
ip neigh
```

ARP is commonly used in local network attacks such as ARP spoofing.

---

# DNS

DNS translates domain names into IP addresses.

Example:

```
google.com

↓

142.250.x.x
```

Useful commands:

```bash
nslookup google.com
```

```bash
dig google.com
```

DNS enumeration is a common step during reconnaissance.

---

# DHCP

DHCP automatically assigns:

- IP Address
- Gateway
- DNS Server
- Subnet Mask

Without DHCP, every device would require manual configuration.

---

# Routing

Routers connect different networks together.

View your routing table:

```bash
ip route
```

Example:

```
default via 192.168.1.1
```

This indicates the default gateway.

---

# ICMP

ICMP is used for diagnostics.

Example:

```bash
ping 192.168.1.1
```

Traceroute also relies on ICMP.

```bash
traceroute google.com
```

# Useful Networking Commands

Display IP addresses:

```bash
ip addr
```

Display routing table:

```bash
ip route
```

Show listening ports:

```bash
ss -tulnp
```

Display active connections:

```bash
netstat -an
```

Check DNS:

```bash
dig google.com
```

Test connectivity:

```bash
ping 8.8.8.8
```

Trace network path:

```bash
traceroute google.com
```

View ARP table:

```bash
arp -a
```

Display interfaces:

```bash
ip link
```

---

# Best Practices

- Learn the most common ports by memory.
- Always identify the subnet before scanning.
- Understand whether a service uses TCP or UDP.
- Verify connectivity before troubleshooting tools.
- Enumerate all exposed services before attempting exploitation.
- Keep notes of discovered hosts and ports.
- Don't assume a host is offline because it doesn't respond to ICMP.

A strong understanding of networking makes enumeration faster and more effective.

---

# Summary

Networking is the foundation of every penetration test.

By understanding IP addressing, ports, protocols, routing, DNS, and TCP/IP, you'll be able to identify targets, map networks, and interact with services confidently.

Most attacks begin with proper network enumeration—and that starts with solid networking knowledge.
