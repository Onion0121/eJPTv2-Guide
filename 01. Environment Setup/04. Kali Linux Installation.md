# Introduction

Kali Linux is the operating system used throughout this guide.

It is the most widely adopted Linux distribution for penetration testing and comes pre-installed with hundreds of security tools used by ethical hackers, penetration testers, incident responders, and security researchers.

Throughout this repository, Kali Linux will be your **attacking machine**.

---

# What is Kali Linux?

Kali Linux is a Debian-based Linux distribution maintained by **Offensive Security**.

It is specifically designed for offensive security tasks such as:

- Penetration Testing
- Vulnerability Assessment
- Digital Forensics
- Reverse Engineering
- Wireless Security Testing
- Web Application Security
- Password Cracking
- Network Analysis

Unlike a standard Linux distribution, Kali already includes most of the tools required during a penetration test.

---

# Why Kali Linux?

There are many Linux distributions capable of running penetration testing tools, but Kali offers several advantages.

## Advantages

- Hundreds of pre-installed security tools
- Regular updates
- Excellent hardware compatibility
- Large community support
- Official documentation
- Frequently used in cybersecurity courses and certifications

For the **eJPT**, Kali Linux is the recommended attacker operating system.

# Downloading Kali Linux

Download Kali Linux only from the official website.

Choose the latest stable release available.

The recommended version for beginners is:

- **64-bit Installer**
- **Virtual Machine Image (VirtualBox or VMware)**

If you are using VirtualBox or VMware, downloading the pre-built virtual machine image will save time.

---

# Choosing the Right Image

Kali provides several download options.

## Installer ISO

Recommended if you want to learn the installation process.

Pros

- Full control
- Custom partitioning
- Better learning experience

---

## Pre-built Virtual Machine

Recommended for beginners.

Pros

- Ready to use
- Faster setup
- VMware Tools or Guest Additions often included
- Less configuration required

---

## Live Boot

Useful for testing Kali without installing it.

Not recommended for daily practice.

# Installing Kali Linux on a Virtual Machine

Create a new virtual machine.

Recommended settings:

| Setting | Value |
|----------|------|
| OS Type | Debian 64-bit |
| CPU | 2 vCPUs |
| RAM | 4096 MB |
| Disk | 40 GB |
| Video Memory | 128 MB |

Enable:

- EFI (optional)
- Bidirectional Clipboard
- Drag & Drop
- Shared Clipboard

If available, enable:

- Nested Paging
- VT-x / AMD-V
- Hardware Virtualization

# First Boot

After installation, log in using your credentials.

Open a terminal and verify your environment.

Check the current user:

```bash
whoami
```

Check the hostname:

```bash
hostname
```

Check your IP address:

```bash
ip addr
```

Check routing:

```bash
ip route
```

Check internet connectivity:

```bash
ping -c 4 8.8.8.8
```

Check DNS resolution:

```bash
ping google.com
```

If all tests succeed, your installation is ready.

# Initial Configuration

Before installing additional tools, configure your system.

Update package repositories:

```bash
sudo apt update
```

Upgrade installed packages:

```bash
sudo apt full-upgrade -y
```

Remove unnecessary packages:

```bash
sudo apt autoremove -y
```

Clean package cache:

```bash
sudo apt autoclean
```

Reboot if necessary:

```bash
sudo reboot
```

# Updating Kali Linux

Keeping Kali updated is important.

Run regularly:

```bash
sudo apt update
sudo apt full-upgrade -y
```

Verify your kernel version:

```bash
uname -r
```

Verify your operating system:

```bash
cat /etc/os-release
```

# Creating Aliases

Aliases reduce typing and speed up your workflow.

Edit:

```bash
nano ~/.bashrc
```

Example aliases:

```bash
alias ll='ls -lah'
alias update='sudo apt update && sudo apt full-upgrade -y'
alias ports='netstat -tulnp'
alias myip='ip addr'
```

Reload:

```bash
source ~/.bashrc
```

# Common Issues

## No Internet Connection

Check:

```bash
ip addr
```

Verify routing:

```bash
ip route
```

Check DNS:

```bash
cat /etc/resolv.conf
```

---

## apt Update Fails

Refresh repositories:

```bash
sudo apt clean
sudo apt update
```

---

## Low Disk Space

Check disk usage:

```bash
df -h
```

Locate large directories:

```bash
du -sh * | sort -h
```

---

## Tool Not Found

Search installed packages:

```bash
which nmap
```

Install missing package:

```bash
sudo apt install package-name
```
