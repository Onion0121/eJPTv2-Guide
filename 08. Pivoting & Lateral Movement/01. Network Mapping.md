# Introduction

Network mapping is the process of discovering and documenting systems, networks, services, and communication paths inside a target environment.

During a penetration test, gaining access to one machine does not always mean the assessment is complete.

The compromised machine may provide access to internal networks that are not directly reachable from the internet.

The objective:

```
Compromised Host

↓

Discover Internal Network

↓

Identify New Targets

↓

Attack Additional Systems
```

---

# What is Network Mapping?

Network mapping involves identifying:

- Network ranges
- Active hosts
- Open ports
- Running services
- Operating systems
- Network routes
- Trust relationships

The information collected helps penetration testers understand the environment and identify possible attack paths.

---

# Why Network Mapping Matters

Many organizations use internal networks that are hidden from external attackers.

Examples:

External network:

```
Internet

↓

Web Server
```

Internal network:

```
Web Server

↓

Database Server

↓

Domain Controller
```

After compromising the web server, an attacker may discover and attack internal systems.

# Identifying Network Interfaces

The first step after obtaining access is checking network configuration.

---

## Linux

View interfaces:

```bash
ip addr
```

Example:

```
eth0
10.10.10.15
```

---

Routing information:

```bash
ip route
```

Example:

```
10.10.10.0/24 dev eth1
```

This reveals accessible networks.

---

## Windows

View interfaces:

```cmd
ipconfig
```

Detailed information:

```cmd
ipconfig /all
```

Look for:

- IP address
- Gateway
- DNS servers
- Domain information

---

# Discovering Internal Networks

After identifying interfaces, determine reachable networks.

Example:

```
Attacker

↓

Compromised Machine

↓

Internal Network
```

Information sources:

- Network interfaces
- Routing tables
- ARP cache
- DNS information

---

## ARP Table

Linux:

```bash
arp -a
```

Windows:

```cmd
arp -a
```

Shows recently contacted devices.

Example:

```
10.10.10.5

10.10.10.20
```

---

# Host Discovery

The goal is finding active systems.

---

## Nmap Host Discovery

Example:

```bash
nmap -sn 10.10.10.0/24
```

This performs a ping scan.

Example output:

```
Host is up

10.10.10.5

10.10.10.20
```

---

## ICMP Discovery

Traditional ping:

```bash
ping 10.10.10.5
```

---

## TCP Discovery

Some hosts block ICMP.

Use:

```bash
nmap -PS80,443 10.10.10.0/24
```

---

# Port Scanning Internal Hosts

After finding active hosts, identify open ports.

Example:

```bash
nmap 10.10.10.5
```

---

Full scan:

```bash
nmap -p- 10.10.10.5
```

---

Service detection:

```bash
nmap -sV 10.10.10.5
```

---

Operating system detection:

```bash
nmap -O 10.10.10.5
```

---

# Service Enumeration

Open ports reveal possible attack paths.

Example:

```
22/tcp SSH

80/tcp HTTP

445/tcp SMB

3306/tcp MySQL
```

| 3389 | RDP |

# Routing Information

Routing tables show where traffic can go.

---

## Linux

```bash
route -n
```

or:

```bash
ip route
```

Example:

```
10.10.20.0/24 via eth1
```

---

## Windows

```cmd
route print
```

Shows:

- Networks
- Gateways
- Interfaces

# Pivoting and Network Access

A compromised machine can act as a bridge.

Example:

```
Attacker

↓

Compromised Host

↓

Internal Server
```

The compromised host allows access to systems that were previously unreachable.

Common pivoting methods:

- SSH tunneling
- Proxychains
- Port forwarding
- SOCKS proxies
- VPN tunnels

---

# Useful Network Mapping Tools

## Nmap

The most common network scanner.

Capabilities:

- Host discovery
- Port scanning
- Service detection
- OS detection

---

## Netdiscover

Used for discovering hosts.

Example:

```bash
netdiscover -r 10.10.10.0/24
```

---

## ARP-scan

Discover local network devices.

Example:

```bash
arp-scan -l
```

---

## CrackMapExec

Useful for Windows environments.

Capabilities:

- SMB enumeration
- Credential testing
- Domain discovery

---

## BloodHound

Maps Active Directory relationships.

Finds:

- Users
- Groups
- Computers
- Privilege paths


# Defense & Mitigation

Protect networks by:

- Implementing network segmentation
- Restricting unnecessary services
- Monitoring internal traffic
- Using firewalls
- Limiting user privileges
- Monitoring lateral movement
- Applying least privilege
