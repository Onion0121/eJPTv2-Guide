# Introduction

WordPress is the world's most popular Content Management System (CMS), powering over 40% of all websites.

Because of its popularity, it is one of the most common targets during penetration tests.

Before attempting exploitation, a penetration tester should enumerate:

- WordPress version
- Installed plugins
- Installed themes
- User accounts
- Login pages
- Backup files
- Interesting directories
- Configuration issues

Proper enumeration often reveals enough information to compromise the application without exploiting complex vulnerabilities.

---

# What is WordPress?

A standard WordPress installation contains:

```
Website

│

├── wp-admin/

├── wp-content/

│ ├── plugins/

│ └── themes/

├── wp-includes/

└── wp-login.php
```

Important directories:

| Directory | Purpose |
|------------|----------|
| wp-admin | Administration panel |
| wp-content | Plugins, themes and uploads |
| wp-includes | Core PHP files |
| wp-login.php | Login portal |

# Identifying WordPress

Common files:

```
/wp-login.php

/xmlrpc.php

/readme.html

/license.txt
```

Example:

```bash
curl http://TARGET/readme.html
```

The readme often discloses the WordPress version.

```bash
curl http://TARGET/robots.txt
```

Readme:

```bash
curl http://TARGET/readme.html
```

XML-RPC:

```bash
curl http://TARGET/xmlrpc.php
```

# Enumerating Users

WordPress usernames are often publicly exposed.

Author ID enumeration:

```
http://TARGET/?author=1
```

Response:

```
/author/admin/
```

The username is:

```
admin
```

Repeat:

```
?author=2

?author=3

?author=4
```

---

RSS feeds may also reveal usernames:

```
/feed
```

---

# Enumerating Themes

Themes are stored in:

```
/wp-content/themes/
```

Example:

```
http://TARGET/wp-content/themes/
```

If directory listing is disabled, inspect page source.

Example:

```html
/wp-content/themes/twentytwentythree/style.css
```

Current theme:

```
twentytwentythree
```

# Enumerating Plugins

Plugins reside in:

```
/wp-content/plugins/
```

Common plugin paths:

```
/wp-content/plugins/contact-form-7/

/wp-content/plugins/woocommerce/

/wp-content/plugins/elementor/
```

Plugins often reveal vulnerable versions.

# WPScan

WPScan is the standard WordPress enumeration tool.

Basic scan:

```bash
wpscan --url http://TARGET
```

Enumerate users:

```bash
wpscan \
--url http://TARGET \
-e u
```

Enumerate plugins:

```bash
wpscan \
--url http://TARGET \
-e p
```

Enumerate themes:

```bash
wpscan \
--url http://TARGET \
-e t
```

Enumerate everything:

```bash
wpscan \
--url http://TARGET \
-e ap,at,u
```

Where:

```
ap = All Plugins

at = All Themes

u = Users
```

# Version Detection

WPScan automatically detects:

- Core version
- Plugin versions
- Theme versions

Manual method:

```bash
curl http://TARGET/readme.html
```

Or inspect source:

```html
<meta name="generator"

content="WordPress 6.4.2">
```

# Common Misconfigurations

Frequently encountered issues:

- Weak admin passwords
- Default usernames
- Exposed XML-RPC
- Outdated plugins
- Outdated themes
- Directory listing enabled
- Public backups
- Exposed configuration files

---

# Defense & Mitigation

Secure WordPress by:

- Updating WordPress regularly
- Removing unused plugins
- Removing unused themes
- Disabling directory listing
- Restricting XML-RPC
- Enforcing strong passwords
- Enabling Multi-Factor Authentication
- Performing regular backups
- Using a Web Application Firewall (WAF)
