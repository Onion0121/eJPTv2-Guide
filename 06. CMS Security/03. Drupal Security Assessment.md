# Introduction

Drupal is one of the most popular Content Management Systems (CMS) used for building dynamic websites and enterprise applications.

Like WordPress, Drupal provides:

- Content management
- User management
- Themes
- Modules
- Database integration

Because Drupal installations often contain third-party modules, outdated versions, and insecure configurations, they are common targets during penetration tests.

Before exploitation, a penetration tester should enumerate:

- Drupal version
- Installed modules
- Themes
- Users
- Configuration files
- Login portals
- Known vulnerabilities

Proper enumeration is usually the key to successful exploitation.

---

# What is Drupal?

Drupal is an open-source CMS written mainly in PHP.

A typical Drupal installation contains:

```
Website

│

├── core/

├── modules/

├── themes/

├── sites/

│   └── default/

│       └── settings.php

├── index.php

└── user/login
```

Important directories:

| Directory | Purpose |
|-----------|---------|
| core/ | Drupal core files |
| modules/ | Installed modules |
| themes/ | Website themes |
| sites/ | Configuration files |
| user/login | Authentication page |

```
Drupal

```


Check headers:

```bash
curl -I http://TARGET
```


# Interesting Files

Always check:

```
CHANGELOG.txt

README.txt

robots.txt

settings.php

install.php

update.php
```

Important configuration file:

```
sites/default/settings.php
```

May contain:

- Database credentials
- Hash salts
- Database hostname
- Configuration information

Example:

```php
$databases['default']['default'] = array(
'database' => 'drupal',
'username' => 'drupal',
'password' => 'password'
);
```

# Droopescan

Droopescan is a CMS enumeration tool.

Install:

```bash
pip3 install droopescan
```

Scan Drupal:

```bash
droopescan scan drupal -u http://TARGET
```

Example output:

```
Version: Drupal 9.4.8

Plugins detected:

views
token
```

# Drupalgeddon

## Drupalgeddon1


Affected:

```
Drupal 7.x
```

Type:

```

```

Impact:

```
Remote Code Execution
```

CVE:

```

```

Affected:

```
Drupal 7.x

```

Type:

```

```

Impact:


```bash
find / -name "*.php" 2>/dev/null
```

```
sites/default/settings.php
```

# Common Misconfigurations

Common problems:

- Outdated Drupal core
- Vulnerable modules
- Weak administrator passwords
- Exposed settings files
- Directory listing enabled
- Unrestricted file uploads
- Poor permissions
- Default accounts

---

# Defense & Mitigation

Secure Drupal by:

- Updating Drupal core
- Updating modules
- Removing unused modules
- Protecting configuration files
- Enforcing strong passwords
- Enabling MFA
- Restricting file uploads
- Using security modules
- Monitoring logs
