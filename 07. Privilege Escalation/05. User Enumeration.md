# Introduction

User enumeration is one of the first steps performed after gaining access to a Windows system.

The objective is to understand:

- Who you are logged in as
- What users exist on the system
- Which users have administrative privileges
- Which accounts may contain valuable information
- How privilege escalation opportunities can be identified

During a penetration test, user enumeration helps identify possible paths from a low-privileged account to administrator or SYSTEM access.

Typical workflow:

```
Initial Access

↓

Identify Current User

↓

Enumerate Users

↓

Identify Privileged Accounts

↓

Find Escalation Path
```

---

# Why User Enumeration Matters

User accounts provide valuable information about the target system.

Attackers look for:

- Administrator accounts
- Service accounts
- Disabled accounts
- Old accounts
- Domain users
- Password reuse opportunities

Examples:

```
Administrator

BackupAdmin

SQLService

WebAdmin
```

These accounts may provide opportunities for:

- Credential attacks
- Privilege escalation
- Lateral movement

# Windows User Accounts

Windows contains different types of accounts.

---

## Local Users

Stored on the local machine.

Database:

```
SAM Registry
```

Examples:

```
Administrator

Guest

User
```

---

## Domain Users

Managed by Active Directory.

Stored in:

```
Active Directory Database
```

Examples:

```
DOMAIN\john

DOMAIN\administrator
```

---

## Service Accounts

Used by applications and services.

Examples:

```
sqlservice

iisuser

backup
```

Service accounts often have excessive privileges.

---

# Identifying Current User

The first command after obtaining access:

```cmd
whoami
```

Example:

```
desktop\john
```

---

Detailed information:

```cmd
whoami /all
```

Shows:

- Username
- SID
- Groups
- Privileges
- Security tokens

---

Check user privileges:

```cmd
whoami /priv
```

Example:

```
SeImpersonatePrivilege
```

Potential privilege escalation opportunity.

---

# Enumerating Local Users

List local users:

```cmd
net users
```

Example:

```
Administrator

Guest

john

backup
```

---

Get information about a user:

```cmd
net user john
```

Example output:

```
Account active
Password last set
Local group memberships
```

---

PowerShell alternative:

```powershell
Get-LocalUser
```

---

# Enumerating Domain Users

If the machine belongs to a domain:

Check domain:

```cmd
systeminfo
```

or:

```cmd
echo %USERDOMAIN%
```

---

Enumerate domain users:

```cmd
net user /domain
```

Example:

```
administrator

john

support
```

---

Query specific domain user:

```cmd
net user administrator /domain
```

---

PowerShell:

```powershell
Get-ADUser -Filter *
```

---

# Enumerating User Groups

Groups determine permissions.

List groups:

```cmd
net localgroup
```

---

Show administrators:

```cmd
net localgroup administrators
```

Example:

```
Administrator

john
```

---

Check current groups:

```cmd
whoami /groups
```

Important groups:

```
Administrators

Remote Desktop Users

Backup Operators

Power Users
```

---

# Finding Administrator Accounts

Administrator accounts are valuable targets.

Check:

```cmd
net localgroup administrators
```

Example:

```
Administrator

DOMAIN\AdminUser
```

---

Domain administrators:

```cmd
net group "Domain Admins" /domain
```

Members of:

```
Domain Admins
```

have complete Active Directory control.

# User Information Gathering

Collect information about users.

Check account status:

```cmd
net user username
```

Information:

- Account active
- Password expiration
- Last login
- Group membership

---

Search usernames:

```cmd
dir C:\Users
```

Example:

```
Administrator

Public

john
```

---

```cmd
C:\Users\
```

Interesting files:

```
Desktop

Documents

Downloads

AppData
```

# Credential Discovery

Users may store credentials in:

- Documents
- Scripts
- Configuration files
- Browser profiles
- Backup files

Search:

```cmd
findstr /si password C:\*.txt
```

---

Search configuration files:

```cmd
dir /s *.config
```

---

Search scripts:

```cmd
dir /s *.bat
```

Look for:

```
username

password

token
```

# Automated Enumeration Tools

## WinPEAS

WinPEAS automatically searches for:

- Users
- Groups
- Credentials
- Services
- Permissions
- Privileges

Execute:

```cmd
winpeas.exe
```

---

## PowerUp

PowerShell privilege escalation tool.

Load:

```powershell
Import-Module .\PowerUp.ps1
```

Run:

```powershell
Invoke-AllChecks
```

---

## BloodHound

Used for Active Directory enumeration.

Finds:

- Users
- Groups
- Relationships
- Privilege paths

# Defense & Mitigation

Secure Windows accounts by:

- Removing unnecessary accounts
- Disabling unused users
- Applying least privilege
- Protecting administrator accounts
- Using strong passwords
- Monitoring account activity
- Limiting domain privileges
- Reviewing group memberships
