# Introduction

Privilege escalation is the process of increasing the level of access obtained on a compromised system.

During a penetration test, attackers usually start with a low-privileged user account and attempt to obtain higher privileges.

On Linux systems, the main goal is usually:

```
Normal User

↓

Root User
```

Obtaining root access allows an attacker to:

- Access sensitive files
- Modify system configurations
- Install persistence mechanisms
- Access other users
- Fully compromise the machine

Privilege escalation is one of the most important phases of a penetration test.

---

# What is Privilege Escalation?

Privilege escalation occurs when a user abuses a vulnerability, misconfiguration, or weak permission to gain additional privileges.

There are two main types:

## Vertical Privilege Escalation

Increasing privileges on the same system.

Example:

```
www-data

↓

root
```

---

## Horizontal Privilege Escalation

Accessing another user account with similar privileges.

Example:

```
user1

↓

user2
```

---

# Linux User Privileges

Linux uses a permission-based security model.

Important users:

## Root User

The root account has complete control.

UID:

```
0
```

Example:

```bash
whoami
```

Output:

```
root
```

---

## Normal Users

Regular accounts have limited permissions.

Example:

```
yassin
```

---

## Service Users

Used by applications.

Examples:

```
www-data

mysql

ssh

ftp
```

These accounts are common starting points during penetration tests.

# Permission Enumeration

Linux permissions:

```
r = read

w = write

x = execute
```

Example:

```
-rwxr-xr-x
```

Meaning:

```
Owner: read/write/execute

Group: read/execute

Others: read/execute
```

---

Find writable files:

```bash
find / -writable -type f 2>/dev/null
```

Writable files executed by root can lead to privilege escalation.

---

# SUID Permissions

SUID allows a program to run with the owner's privileges.

Find SUID binaries:

```bash
find / -perm -4000 2>/dev/null
```

Example:

```
/usr/bin/passwd
```

Potentially dangerous binaries:

```
vim

python

find

bash

nmap
```

Search known exploits:

```bash
searchsploit suid
```

---

# Sudo Privileges

Sudo allows users to execute commands with elevated privileges.

Check:

```bash
sudo -l
```

Example:

```
(user) NOPASSWD: /usr/bin/vim
```

This may allow root access.

GTFOBins:

```
https://gtfobins.github.io/
```

---

# Cron Jobs

Cron executes scheduled tasks.

List cron jobs:

```bash
cat /etc/crontab
```

Check:

```bash
ls -la /etc/cron*
```

Look for:

- Scripts executed by root
- Writable files
- Weak permissions

Example:

```
root /backup.sh
```

If `/backup.sh` is writable:

```
Modify script

↓

Wait execution

↓

Obtain root
```

---

# Linux Capabilities

Capabilities provide specific privileges to programs.

Check:

```bash
getcap -r / 2>/dev/null
```

Example:

```
python3 cap_setuid+ep
```

Capabilities can sometimes bypass normal restrictions.

# Kernel Exploits

- Dirty Pipe

# Defense & Mitigation

Protect Linux systems by:

- Updating the kernel
- Removing unnecessary SUID binaries
- Restricting sudo permissions
- Applying least privilege
- Monitoring cron jobs
- Securing file permissions
- Removing unnecessary services
- Auditing user accounts
- Monitoring logs
