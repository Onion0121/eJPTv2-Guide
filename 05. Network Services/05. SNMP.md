# Introduction

Simple Network Management Protocol (SNMP) is widely used to monitor and manage network devices such as routers, switches, printers, firewalls, servers, and IoT devices.

Because administrators often leave the default community strings unchanged, SNMP can expose a large amount of valuable information without requiring authentication.

SNMP enumeration is a common task during penetration tests and appears regularly in **eJPT** labs.

---

# What is SNMP?

SNMP allows administrators to remotely monitor and configure network devices.

Typical devices using SNMP include:

- Routers
- Switches
- Firewalls
- Printers
- Linux Servers
- Windows Servers
- NAS Devices
- IP Cameras

SNMP exposes device information through a hierarchical database called the **Management Information Base (MIB)**.

---

# SNMP Versions

| Version | Authentication | Encryption |
|----------|---------------|------------|
| SNMPv1 | Community String | ❌ |
| SNMPv2c | Community String | ❌ |
| SNMPv3 | Username + Authentication | ✅ |

During the **eJPT**, most targets use:

- SNMPv1
- SNMPv2c

These are much easier to enumerate.

# Default Ports

| Port | Protocol | Service |
|------|----------|---------|
| **161** | UDP | SNMP |
| **162** | UDP | SNMP Trap |

Verify:

```bash
nmap -sU -p161 TARGET
```

# Why SNMP Matters

SNMP may reveal:

- Hostname
- Operating System
- Installed Software
- Running Services
- Network Interfaces
- IP Addresses
- Routing Tables
- Usernames
- Device Descriptions
- Uptime
- Processes

Sometimes it even exposes passwords or configuration files.

# Identifying SNMP

Basic UDP scan:

```bash
nmap -sU -p161 TARGET
```

Version detection:

```bash
nmap -sU -sV -p161 TARGET
```

Aggressive scan:

```bash
nmap -A -sU -p161 TARGET
```

---

# Enumerating SNMP with Nmap

Useful NSE scripts:

```bash
nmap --script snmp-info TARGET
```

Run all SNMP scripts:

```bash
nmap --script "snmp*" TARGET
```

Example:

```
Hostname: Server01

SNMP Version: v2c

Contact: Administrator

Location: Data Center
```

# SNMPWalk

The most commonly used SNMP enumeration tool.

Using the default community string:

```bash
snmpwalk -v2c -c public TARGET
```

Specific OID:

```bash
snmpwalk -v2c -c public TARGET 1.3.6.1.2.1.1
```

System information:

```bash
snmpwalk -v2c -c public TARGET system
```

Network interfaces:

```bash
snmpwalk -v2c -c public TARGET interfaces
```

Processes:

```bash
snmpwalk -v2c -c public TARGET hrSWRun
```

Installed software:

```bash
snmpwalk -v2c -c public TARGET hrSWInstalled
```

# SNMPCheck

Another useful enumeration tool:

```bash
snmp-check TARGET
```

Specify the community string:

```bash
snmp-check TARGET -c public
```

```

Interfaces

Routing Table

Processes

Installed Software

Storage Devices

Users
```

# Common Mistakes

## Forgetting UDP Scans

Many beginners only scan TCP ports.

SNMP uses UDP.

Correct scan:

```bash
nmap -sU -p161 TARGET
```

---

## Testing Only One Community String

Do not stop after trying:

```
public
```

Always test additional defaults such as:

```
private

manager

admin
```

---

## Ignoring Installed Software

SNMP often reveals installed software.

This helps identify:

- Apache
- IIS
- MySQL
- Tomcat
- FTP Servers
- Backup Software

These may contain known vulnerabilities.

---

## Skipping Interface Enumeration

Network interfaces may reveal:

- Internal IP addresses
- VLANs
- Multiple network adapters
- Additional attack paths

---

# Defense & Mitigation

Secure SNMP by:

- Disabling SNMP if unnecessary
- Using SNMPv3 instead of v1/v2c
- Changing default community strings
- Restricting SNMP access with firewalls
- Monitoring SNMP requests
- Limiting management access to trusted hosts
- Keeping network devices updated
