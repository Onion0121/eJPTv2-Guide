# Introduction

Remote Procedure Call (RPC) is a protocol used by Windows systems to allow remote management and communication between machines.

For penetration testers, **RPC enumeration** is an excellent way to discover:

- Domain users
- Local users
- Groups
- Password policies
- Security Identifiers (SIDs)
- Domain information

RPC is frequently available on Windows systems and often appears in the **eJPT** exam.

---

# What is RPC?

RPC (Remote Procedure Call) allows one computer to execute functions on another computer over the network.

Windows uses RPC for:

- Active Directory
- User management
- Group management
- Authentication
- Remote administration

RPC commonly works together with SMB.

---

# Why RPC Matters

A successful RPC connection may reveal:

- Usernames
- Groups
- Domain names
- Domain SID
- Password policies
- User descriptions
- Account information

This information is extremely valuable before performing password attacks.

# Checking RPC Availability

First, verify that SMB ports are open.

```bash
nmap -sV -p139,445 TARGET
```

If SMB is available, RPC is usually accessible.

---

# Connecting with rpcclient

Anonymous connection:

```bash
rpcclient -U "" TARGET
```

If prompted:

```
Password:

Press ENTER
```

If authentication succeeds:

```
rpcclient $
```

You now have an interactive RPC shell.

# Anonymous Enumeration

Some systems allow guest or anonymous enumeration.

Useful command:

```
enumdomusers
```

Example:

```
user:[Administrator]

user:[Guest]

user:[John]

user:[Backup]
```

Groups:

```
enumdomgroups
```

Aliases:

```
enumalsgroups builtin
```

Domain information:

```
querydominfo
```

Password policy:

```
getdompwinfo
```

---

# Authenticated Enumeration

If credentials are available:

```bash
rpcclient \
-U administrator \
TARGET
```

Or:

```bash
rpcclient \
-U DOMAIN\\john \
TARGET
```

Authenticated access usually provides much more information.

# Useful RPC Commands

List domain users:

```
enumdomusers
```

List groups:

```
enumdomgroups
```

Builtin groups:

```
enumalsgroups builtin
```

Display domain information:

```
querydominfo
```

Display password policy:

```
getdompwinfo
```

Lookup usernames:

```
lookupnames administrator
```

Lookup SID:

```
lookupsids S-1-5-21-...
```

Query a specific user:

```
queryuser RID
```

Query a specific group:

```
querygroup RID
```

List available shares:

```
netshareenum
```

Exit:

```
quit
```

---

# RID Cycling

Every Windows account has a Relative Identifier (RID).

Example:

```
500

Administrator

501

Guest

1000+

Normal Users
```

Retrieve users:

```
queryuser 500
```

```
queryuser 501
```

```
queryuser 1000
```

Continue increasing the RID until no additional users are found.

RID cycling is an excellent technique for discovering hidden accounts.

# Combining RPC with Other Tools

RPC works best alongside other enumeration tools.

Enum4linux:

```bash
enum4linux -a TARGET
```

CrackMapExec:

```bash
crackmapexec smb TARGET --users
```

Nmap:

```bash
nmap --script smb* TARGET
```

SMB shares:

```bash
smbclient -L //TARGET/ -N
```

The combination of these tools provides a complete picture of the target.

# Common Mistakes

## Ignoring Anonymous Access

Always test:

```bash
rpcclient -U "" TARGET
```

Many beginner labs intentionally allow guest enumeration.

---

## Forgetting Password Policy

Retrieve the password policy:

```
getdompwinfo
```

It may reveal:

- Minimum length
- Complexity requirements
- Password age

This helps optimize password attacks.

---

## Ignoring User Descriptions

Some environments store useful information inside user descriptions.

Example:

```
John Smith

IT Administrator

Temporary Password: Welcome123
```

Always inspect user information.

---

## Not Combining Tools

RPC should not be used alone.

Combine it with:

- SMB Enumeration
- Enum4linux
- CrackMapExec
- Hydra
- Nmap

---

# Defense & Mitigation

Protect RPC services by:

- Restricting anonymous enumeration
- Disabling guest accounts
- Limiting RPC exposure with firewalls
- Using strong authentication
- Applying the principle of least privilege
- Monitoring remote administration activity
- Keeping Windows systems updated
