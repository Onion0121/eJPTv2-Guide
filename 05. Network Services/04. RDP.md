# Introduction

Remote Desktop Protocol (RDP) is Microsoft's proprietary protocol for remotely accessing Windows systems through a graphical interface.

RDP is commonly found in enterprise environments and frequently appears in penetration testing engagements. Weak credentials, exposed RDP services, and poor security configurations often provide an easy path to compromise.

The **eJPT** exam may require you to enumerate RDP services, perform password attacks, or authenticate using credentials discovered elsewhere.

---

# What is RDP?

RDP allows remote graphical access to Windows systems.

Typical uses include:

- Remote administration
- Remote support
- Server management
- Virtual Desktop Infrastructure (VDI)

Once authenticated, users receive a full Windows desktop session.

| Port | Protocol | Service |
|------|----------|---------|

Verify the service:


# Identifying RDP

Basic scan:

```bash
nmap -sV -p3389 TARGET
```

Example:

```
3389/tcp open ms-wbt-server
```

OS detection:

```bash
nmap -A TARGET
```

---

# Nmap Enumeration

Determine encryption:

```bash
nmap --script rdp-enum-encryption TARGET
```

Retrieve NTLM information:

```bash
nmap --script rdp-ntlm-info TARGET
```

Check for BlueKeep:

```bash
nmap --script rdp-vuln-ms12-020 TARGET
```

Run all RDP-related scripts:

```bash
nmap --script "rdp*" TARGET
```

# Banner Enumeration

Obtain additional information:

```bash
nmap -sV -A -p3389 TARGET
```

Possible output:

```
Windows Server 2019

Network Level Authentication (NLA): Enabled

TLS Supported
```

Useful information includes:

- Windows version
- Domain name
- Computer name
- Encryption methods
- NLA status

# Connecting to RDP

Using **xfreerdp**:

```bash
xfreerdp /u:administrator /p:Password123 /v:TARGET
```

Domain authentication:

```bash
xfreerdp /u:DOMAIN\\john /p:Password123 /v:TARGET
```

Ignore certificate warnings:

```bash
xfreerdp /u:administrator /p:Password123 /v:TARGET /cert:ignore
```

If successful, you will receive a graphical Windows desktop.

# Password Attacks

Hydra example:

```bash
hydra \
-l administrator \
-P /usr/share/wordlists/rockyou.txt \
rdp://TARGET
```

Using a username list:

```bash
hydra \
-L users.txt \
-P rockyou.txt \
rdp://TARGET
```

CrackMapExec:

```bash
crackmapexec rdp TARGET \
-u administrator \
-p Password123
```

Always perform password attacks only when authorized.

# BlueKeep Detection

BlueKeep (CVE-2019-0708) is a critical RDP vulnerability affecting older Windows versions.

Search:

```text
search bluekeep
```

Scanner:

```text
use auxiliary/scanner/rdp/cve_2019_0708_bluekeep
```

Configure:

```text
set RHOSTS TARGET
```

Execute:

```text
run
```

> **Note:** Modern Windows systems are generally patched. On the eJPT exam, BlueKeep is more likely to be a topic of recognition than active exploitation.

# Common Mistakes

## Ignoring Credential Reuse

Credentials recovered from:

- SMB
- FTP
- Web applications
- Databases

may also work for RDP.

Always test reused credentials.

---

## Forgetting NLA

Many systems require **Network Level Authentication (NLA)**.

Older RDP clients may fail to connect even with valid credentials.

---

## Ignoring Domain Names

Corporate environments often require:

```
DOMAIN\username
```

instead of:

```
username
```

---

## Skipping Enumeration

Always determine:

- Windows version
- Encryption
- NLA status
- Domain information

before attempting authentication.

---

# Defense & Mitigation

Secure RDP by:

- Enabling Network Level Authentication
- Using strong passwords
- Enforcing Multi-Factor Authentication
- Restricting RDP via firewall
- Allowing access only through VPN
- Disabling unused RDP services
- Monitoring failed login attempts
- Keeping Windows fully patched

# Summary

RDP is one of the most common remote administration services found on Windows systems. Proper enumeration, combined with credential attacks and credential reuse, often leads to successful access.

In this chapter, you learned how to:

- Identify RDP services
- Enumerate encryption and NTLM information
- Connect using FreeRDP
- Perform password attacks with Hydra
- Use Metasploit RDP modules
- Detect BlueKeep
- Perform basic Windows post-exploitation

Mastering RDP enumeration is an important skill for the **eJPT** exam and provides a strong foundation for Windows penetration testing.
